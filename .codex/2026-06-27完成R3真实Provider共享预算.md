## 结论

R3 已完成基础闭环：

- `RequestExecutionBudget` 提供总 deadline、总尝试次数、剩余时间和停止原因。
- `ProviderAttemptExecutor` 在每次候选下游尝试前扣减共享尝试预算，耗尽时返回明确错误，不继续切换 provider。
- `CreateOpenAICompatibleAttemptHandler()` 把剩余 deadline 截断到 `HttpRequestOptions` 的 connect/send/recv/total timeout，再进入 HTTP I/O。
- `tests/test_ai_gateway_real_provider.cc` 覆盖 retry + failover 不超过总尝试预算，以及 adapter 传入 HTTP 层的 timeout 被剩余 deadline 截断。
