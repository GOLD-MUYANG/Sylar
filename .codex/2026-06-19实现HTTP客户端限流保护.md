# 2026-06-19 实现 HTTP 客户端限流保护

## 目标

根据 `文档/接下来要做什么.md` 第六步，实现 HTTP 客户端并发数限制，先覆盖全局、服务和 Endpoint 三个维度，不做 QPS/token bucket。

## 使用过的命令和原因

1. `sed -n '1,260p' 文档/接下来要做什么.md`
   - 确认第六步的具体要求，以及前五步已经完成的能力边界。

2. `sed -n '1,260p' sylar/http/http_load_balance_client.h`
   - 查看负载均衡客户端已有接口，判断限流应该如何接入。

3. `sed -n '1,360p' sylar/http/http_load_balance_client.cc`
   - 查看 Endpoint 选择和活跃请求计数逻辑，确认请求开始和结束的位置。

4. `sed -n '1,360p' tests/test_http_load_balance_client.cc`
   - 查看现有负载均衡测试，避免继续把所有限流测试都塞进同一个测试文件。

5. `rg -n "FiberSemaphore|Semaphore|active_request|HttpResult::Error|LoadBalance" sylar tests CMakeLists.txt`
   - 查找已有并发控制、错误码和负载均衡相关实现，避免重复造不必要的接口。

6. `cmake -S . -B build`
   - 新增 CMake 源文件和测试目标后重新生成构建目录。

7. `cmake --build build --target test_http_concurrency_limiter`
   - 构建新增限流器单测，确认独立限流器文件边界可编译。

8. `./bin/test_http_concurrency_limiter`
   - 验证限流器单元行为：不限流、服务限流、Endpoint 限流、全局限流。

9. `cmake --build build --target test_http_load_balance_client`
   - 构建负载均衡客户端集成测试，确认限流接入不破坏已有负载均衡逻辑。

10. `./bin/test_http_load_balance_client`
    - 普通沙箱下失败，原因是本地 socket 创建返回 `Operation not permitted`；这和项目已有本地 socket 测试限制一致。

11. `./bin/test_http_load_balance_client`（允许本地 socket 后）
    - 验证负载均衡客户端集成测试通过，Endpoint 并发限制会返回 `RATE_LIMITED`。

12. `cmake --build build`
    - 按项目验证流程完整构建，确认新增源码和测试目标不会破坏整体构建。

## 结果

- 新增 `sylar/http/http_concurrency_limiter.h/.cc`，限流能力从负载均衡客户端中拆出。
- `HttpLoadBalanceClient` 只负责接入限流器，不承载具体计数实现。
- 新增 `tests/test_http_concurrency_limiter.cc` 覆盖限流器单元行为。
- `tests/test_http_load_balance_client.cc` 只保留一条限流集成测试，避免继续膨胀。
