```text

status=request_failed attempts=1 http_status=502 message="real provider smoke failed: 上游 Provider 当前不可用"
```

## 排查命令与原因

1. `sed -n '1,90p' scripts/smoke_ai_gateway_real_provider.sh`
   - 确认 smoke 脚本当前设置的 `base_url`、`chat_path`、`upstream_model`、`api_key_env` 和超时时间。

2. `nl -ba sylar/socket.cc | sed -n '190,240p'`
   - 定位日志中 `<sylar/socket.cc:222>` 的含义。
   - 结论：该日志来自 `Socket::connect()` 的 `connect_with_timeout()` 分支，表示 TCP connect 在指定超时时间内没有完成。

3. `rg -n "connect\\(|Operation now in progress|timeout|CONNECT_FAIL|上游 Provider 当前不可用|Provider 当前不可用|DoProviderHttpPost|DoRequest|HttpResult" sylar modules tests`

   - 结论：连接失败会进入 Provider 错误分类，最终脱敏成 `上游 Provider 当前不可用`。

   - 确认请求体和 URL 拼接位置。
   - 结论：模型字段写入 JSON body 的 `model`；本次失败发生在 connect 阶段，还没到 HTTP 请求体、鉴权或模型校验。

5. `nl -ba modules/ai_gateway/real_provider_gateway.cc | sed -n '148,310p'`
   - 确认 `502` 和中文错误信息的来源。
   - 结论：`上游 Provider 当前不可用` 是对连接类失败的对外脱敏消息，不代表 provider 已经返回 502。

6. `git diff -- scripts/smoke_ai_gateway_real_provider.sh 文档/真实提供商网关.md`

## 当前判断

这次失败的直接表现是 TCP connect 返回 `EINPROGRESS` 并被上层当成失败。后续用户确认 `14.103.169.114:443` 可通过 `/dev/tcp` 和 `curl --noproxy '*'` 直连，说明整体公网链路可用。

根因是 smoke 可执行文件原本直接在普通 `main()` 中执行真实 Provider attempt：`Socket::newSock()` 把 fd 纳入 `FdMgr` 后会设为非阻塞，但当前线程没有进入 `IOManager/Scheduler::run()`，hook 未开启；`connect_with_timeout()` 因 hook 未开启直接调用原始 `connect_f()`，非阻塞 fd 返回 `EINPROGRESS`，被 `Socket::connect()` 记录成连接失败。

已修复：`RunRealProviderSmoke()` 在配置和 Key 校验通过后，若当前不在 `IOManager` 中，会创建临时 `IOManager("real-provider-smoke")` 并把真实 provider attempt 调度进去执行；回归测试覆盖 provider attempt 中 `IOManager::GetThis() != nullptr` 且 `is_hook_enable() == true`。

后续又发现一个构建易错点：`scripts/smoke_ai_gateway_real_provider.sh` 原本只在二进制不存在时构建，源码修改后不会自动刷新 `bin/ai_gateway_real_provider_smoke`。已把脚本改为每次运行前都执行 `cmake --build build --target ai_gateway_real_provider_smoke`，避免继续运行旧二进制。
