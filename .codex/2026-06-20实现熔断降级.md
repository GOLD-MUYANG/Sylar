根据 `文档/接下来要做什么.md` 的第七步，实现 HTTP 客户端侧的 Endpoint 维度熔断保护，并接入 `HttpLoadBalanceClient`。

   - 确认当前工作区起点干净，并了解最近已经完成限流、负载均衡、重试等前置能力。

3. `sed -n '1,260p' sylar/http/http_load_balance_client.h`、`sed -n '1,360p' sylar/http/http_load_balance_client.cc`

   - 参考限流器的参数对象、Create()、guard 和独立测试文件边界。

5. `sed -n '1,260p' sylar/http/http_connection.h`、`sed -n '300,620p' sylar/http/http_client.cc`
   - 查看 `HttpResult::Error` 和 `HttpClient::NormalizeResult()` / `ShouldRetry()`，确认熔断失败分类要基于归一化后的结果。

6. `cmake --build build --target test_http_circuit_breaker`
   - 先写 `tests/test_http_circuit_breaker.cc` 后构建，确认新测试目标还不存在。

   - 重新生成构建系统，确认 RED 阶段失败在缺少 `sylar/http/http_circuit_breaker.cc`，说明生产代码尚未实现。

8. `cmake --build build --target test_http_circuit_breaker`、`./bin/test_http_circuit_breaker`
   - 实现熔断器后验证纯状态机测试通过。

9. `cmake --build build --target test_http_load_balance_client`
   - 写负载均衡接入测试后确认 RED 阶段失败在 `HttpLoadBalanceClient::Create()` 缺少熔断参数。

10. `cmake --build build --target test_http_load_balance_client`、`./bin/test_http_load_balance_client`
    - 接入熔断器后构建通过；普通沙箱运行本地 socket 测试失败，错误为 `Operation not permitted`。

11. `./bin/test_http_load_balance_client`
    - 放行本地 127.0.0.1 socket 后重跑通过。

12. `./bin/test_http_circuit_breaker`、`./bin/test_http_concurrency_limiter`、`cmake --build build`
    - 回归新增熔断器测试、相邻限流器测试和全量构建，均通过。

13. `cmake --build build --target test_http_circuit_breaker`
    - 自审发现默认启用熔断会改变旧的 `HttpLoadBalanceClient::Create()` 调用行为，所以先新增 `enabled` 兼容性测试，确认 RED 阶段失败在缺少 `HttpCircuitBreakerOptions::enabled` 字段。

14. `cmake --build build --target test_http_circuit_breaker`、`./bin/test_http_circuit_breaker`
    - 加入 `enabled` 开关后，确认默认关闭熔断；显式 `enabled = true` 的测试覆盖状态转换。

15. `cmake --build build --target test_http_load_balance_client`、`./bin/test_http_load_balance_client`
    - 确认负载均衡接入测试在显式启用熔断后仍通过。

16. `cmake --build build --target test_http_load_balance_client`、`./bin/test_http_load_balance_client`
    - 根据后续讨论补充客户端内部自动 failover 语义：先新增熔断 failover 和限流 failover 测试，确认 RED 阶段分别返回 `CIRCUIT_OPEN` 和 `RATE_LIMITED`。

17. `cmake --build build --target test_http_load_balance_client`、`./bin/test_http_load_balance_client`
    - 调整 `HttpLoadBalanceClient::request()`，同一次请求内记录已尝试的 endpoint key；当前 endpoint 被限流、熔断打开或出现可重试失败时，继续尝试其他未尝试过的 UP endpoint，测试通过。

## 结果

- 新增 `sylar/http/http_circuit_breaker.h` 和 `sylar/http/http_circuit_breaker.cc`。
- 新增 `tests/test_http_circuit_breaker.cc` 并接入 CMake。
- `HttpLoadBalanceClient` 支持可选 `HttpCircuitBreakerOptions`。
- 熔断默认关闭，旧调用保持兼容；调用方设置 `enabled = true` 后启用。
- 多实例客户端支持同一次请求内自动 failover；有备选 endpoint 时不会因为单个 endpoint 限流、熔断或可重试失败就直接返回。
- 新增 `CIRCUIT_OPEN` 错误码。
- 更新 `文档/接下来要做什么.md`，将第七步标记为已完成。
