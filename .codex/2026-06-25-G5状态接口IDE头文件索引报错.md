# G5 状态接口 IDE 头文件索引报错

## 背景

IDE 在 `modules/ai_gateway/ai_gateway_status_servlet.cc` 中提示：

`No member named 'getStatusSnapshots' in 'sylar::http::HttpLoadBalanceClient'`

## 排查命令

1. `cmake --build build --target test_ai_gateway_status`
   - 目的：确认 G5 状态接口测试目标是否能在真实编译链路中通过。
   - 结果：目标可构建。

2. `cmake --build build --target ai_gateway_module`
   - 目的：确认模块动态库是否能编译 `ai_gateway_status_servlet.cc`。
   - 结果：目标可构建。

3. `cmake --build build --target ai_gateway_module --clean-first`
   - 目的：避免旧对象文件掩盖问题，强制重编 `ai_gateway_status_servlet.cc`。
   - 结果：`ai_gateway_status_servlet.cc` 真实编译通过。

4. `rg -n "getStatusSnapshots|HttpEndpointStatusSnapshot" /home/muyang/workspace/sylar/sylar/http/http_load_balance_client.h /home/muyang/workspace/sylar/.worktrees/ai-gateway-g3/sylar/http/http_load_balance_client.h`
   - 目的：确认 IDE 是否可能解析到主仓库旧头文件。
   - 结果：只有 worktree 里的 `http_load_balance_client.h` 有 `getStatusSnapshots()`；主仓库旧头文件没有该成员。

5. `cmake -S . -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON`
   - 目的：为 worktree 生成准确的编译数据库。
   - 结果：生成 `build/compile_commands.json`。

6. `ln -sf build/compile_commands.json compile_commands.json`
   - 目的：让 clangd/IDE 在 worktree 根目录优先读取正确的编译数据库。

7. `cmake --build build --target test_ai_gateway_status && ./bin/test_ai_gateway_status`
   - 目的：验证生成编译数据库后功能目标仍正常。
   - 结果：目标构建和测试运行通过。

## 结论

源码真实编译链路没有 `getStatusSnapshots()` 缺失问题。该 IDE 报错的根因是 worktree 嵌在主仓库目录下，IDE/LSP 很可能把 include 根目录解析到主仓库旧头文件。已在 worktree 根目录放置 `compile_commands.json` 指向 `build/compile_commands.json`，用于让 IDE 使用 worktree 自己的头文件和编译参数。
