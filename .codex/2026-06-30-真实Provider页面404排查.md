# 2026-06-30 真实Provider页面404排查

## 问题

`scripts/demo_ai_gateway_real_provider.sh` 启动后浏览器打开 `/demo` 返回
`404 Not Found`。

## 排查命令和原因

- `sed -n '1,160p' examples/ai_gateway_conf/ai_gateway.yml`
  - 确认当前配置中 `ai_gateway.enabled: false`、`real_providers.enabled: true`。
- `sed -n '220,390p' modules/ai_gateway/ai_gateway_module.cc`
  - 确认 `/demo` 只会在 mock demo trace 或 real demo enabled 时注册；
    如果 real provider 配置加载失败且 mock 也关闭，模块会直接 disabled，
    HTTP Server 仍启动但 `/demo` 不存在。
- `sed -n '160,280p' modules/ai_gateway/real_provider_runtime.cc`
  - 确认 `real_providers.enabled: true` 时，每个 `enabled: true` 的 provider
    都必须能从当前 shell 读到对应 `api_key_env`。
- `rg -n "curl .*demo|真实 Provider 演示页面" scripts/demo_ai_gateway_real_provider.sh`
  - 确认脚本之前没有使用 `curl --fail`，会把 HTTP 404 当成启动成功。

## 结论

当前只测 DeepSeek 时，Ark provider 仍 `enabled: true` 会要求
`ARK_API_DOUBAO` 也存在；如果缺少这个环境变量，real provider 配置加载失败，
`/demo` 不会注册，页面返回 404。

## 修复

- `examples/ai_gateway_conf/ai_gateway.yml`
  - 将 `volcengine-ark.enabled` 改为 `false`，只启用 DeepSeek。
- `scripts/demo_ai_gateway_real_provider.sh`
  - 将 `/demo` 探活改为 `curl --fail`，让 HTTP 404 不再被误判为成功。
  - 失败时提示检查前面的网关日志和 enabled provider 的 `api_key_env`。

- 静态检查脚本包含 `curl --fail`
- 静态检查 DeepSeek 演示配置中 Ark 没有继续 enabled
- `cmake --build build --target bin_sylar`
