# Linux 常用命令学习手册设计

## 1. 目标

从 `.codex/` 中的真实排障记录提取 Linux 命令，整理成面向初学者的通用教程。手册不照搬一次性排障步骤，而是解释命令解决什么问题、如何组合以及如何安全练习。

## 2. 读者和范围

- 目标读者：能打开 Linux 终端，但对命令、参数、管道和重定向不熟悉的初学者。
- 主体素材：`.codex/*.md` 中实际出现过的命令。
- 允许补充：为建立完整学习链路所必需的少量基础命令，如 `pwd`、`cd`、`ls`、`cp`、`mv`、`mkdir` 和 `man`。
- 不包含：完整 Linux 系统管理、发行版安装、Shell 脚本编程大全和 Sylar 业务实现说明。

## 3. 文档结构

正文输出到 `文档/Linux常用命令学习手册.md`，按使用任务而不是字母顺序组织：

1. 命令行、参数和路径基础。
2. 获取帮助和确认命令来源。
3. 目录导航与文件管理。
4. 查看文件和搜索文本。
5. 管道、重定向和命令串联。
6. 文件权限和脚本执行。
7. 进程、信号与端口排查。
8. HTTP、网络和 TLS 检查。
9. CMake 编译与 CTest 测试。
10. Git 只读检查命令。
11. 环境变量和当前 Shell 会话。
12. 危险操作、通用排障流程、练习题和速查表。

## 4. 单个命令的讲解模板

根据命令复杂度选用以下字段，避免为简单命令堆叠无用内容：

1. 解决什么问题。
2. 基本格式。
3. 本手册用到的常见参数。
4. 可安全执行的通用示例。
5. 来自 Sylar 排障记录的实际示例。
6. 输出解读、常见误区或安全提示。

## 5. 命令处理规则

- 合并只是路径、文件名或测试目标不同的重复命令。
- 保留能展示一个新知识点的组合命令。
- 将 Sylar 绝对路径改写为从仓库根目录执行的相对路径，必须保留绝对路径语义时再单独解释。
- 区分 Linux 通用命令、Git/CMake 等开发工具命令和 Sylar 项目自带脚本。
- 不使用真实 API Key、用户私有路径或可能误导读者的过期输出。

## 6. 安全边界

- 默认示例为只读或对临时练习目录可恢复的操作。
- `kill`、`chmod`、`mv`、覆盖重定向、`git reset` 等操作必须标注影响范围和安全替代方案。
- 不提供针对宽泛目录的递归删除示例。
- 将访问外网、占用端口或需要额外依赖的练习标为“环境相关”。

## 7. 验收标准

- `.codex/` 中具有通用学习价值的命令类型均有归类，明显重复项已合并。
- 初学者能从示例中看懂命令、选项、参数、路径、管道和重定向的边界。
- 每个复杂命令至少有一个可复制的安全示例，项目示例的路径与当前仓库一致。
- 文档明确标注高风险操作，不将破坏性命令包装成普通练习。
- Markdown 标题、代码块、表格和内部路径引用格式正确。
